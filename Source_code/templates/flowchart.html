Project Flow

Comprehensive overview of the music genre classification project workflow, from data preprocessing to model evaluation.

The Music Genre Classification Project involves several key steps to process and classify music. It begins with preprocessing the GTZAN Music Genre Dataset, where audio tracks are split into training, validation, and test sets. The features are then extracted using MFCCs (Mel-frequency cepstral coefficients), which capture the audio's spectral characteristics. A Convolutional Neural Network (CNN) is used for classification, where the model learns patterns from the extracted features. The model is trained with a batch size of 32 and 100 epochs, and evaluated on the test set using metrics like accuracy, precision, recall, and F1-score. The final results show good accuracy in classifying music genres, with opportunities for further improvements like data augmentation and more complex models.

Process Overview

The Mel-frequency cepstral coefficients (MFCCs) are extracted using the following steps:

  • Audio file loading using Librosa
  • Pre-emphasis filtering
  • Frame segmentation
  • MFCC computation using DCT

Model Architecture

CNN Layer Configuration

  • Dense Layer (128 neurons)
